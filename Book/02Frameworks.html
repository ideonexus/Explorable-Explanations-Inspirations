Nothing in the world of information technology today is produced in a vacuum. Everything we create is built on the thousands-years foundation of inventions that has come before us. In software development, we can develop rapidly if we develop in an environment where many of the common problems are already solved. Programming languages solve many basic problems. They let us round() numbers, get an .indexOf() a substring() in a string, replace() characters in a string, or perform otherwise complex operations on a Date().

While programming languages provide broadly-used functionality, these functions are basic and are meant to meet common needs when dealing with math, text, or date function. Taking the abstraction further, software frameworks provide a core solution for building an application of a specific type. For example, I think JavaScript has matured into a fantastic programming language, but for web design the jQuery framework makes manipulating HTML and building web applications much easier--but frameworks can be even more specific to a domain. Today there are frameworks to empower building all sorts of amazing applications. The frameworks listed here are only a small sample and are generalized enough to be useful in a wide variety of domains.

Chatbot Frameworks

In 1950, Alan Turning, father of theoretical computer science and artificial intelligence, proposed a method for evaluating the ability of a computer to exhibit intelligence equivalent to that of a human being (Beavers, 2013). Known today as the Turing Test, the method involved a user participating in a double-blind conversation with both a human and a computer. The computer program passes the test if the evaluator cannot reliably tell which is the human and which the computer (Turing, 1950). Since 1991, the--notably controversial--Loebner Prize has held an annual contest evaluating software applications based on how well they perform on the Turing Test.

The Loebner Prize is controversial because the Turing Test is a controversial measure. Philosopher John Searle proposed a thought experiment known as the Chinese Room, where an English-speaker sits in a room, receiving instructions from a Chinese-speaker outside the room. The English-speaker, knowing nothing of Chinese, reviews the written symbols, consults a library of instructions, and responds with Chinese ideograms as the instructions indicate without knowing anything about the content of the inputs or outputs (Searle, 1980). The English-speaker behaves like a computer, but we would not describe the behavior as intelligent.

Many winners of the Loebner Prize work exactly like the human being in the Chinese Room. I have spent many hours working with the 2000, 2001, and 2004 winner, the Artificial Linguistic Internet Computer Entity (A.L.I.C.E.), created by Dr. Richard S. Wallace, uses Artificial Intelligence Markup Language (AIML), another *ML like XML, HTML or XHTML, which looks for a <pattern> in user-inputs and outputs the <template> like so:

        <category>
            <pattern>JULES VERNE</pattern>
            <template>I loved A Trip to the Moon.</template>
        </category>
        <category>
            <pattern>ASIMOV</pattern>
            <template>The Foundation Trilogy was the fall of the Roman Empire on a galactic scale.</template>
        </category>

Alice was effective because it had thousands of these entries, but it doesn't know what it's talking about. We call programs like these Chatbots, and they are growing more sophisticated all the time. Microsoft, Facebook, and even Disney have released platforms for developing them, primarily for providing customer support without the cost of hiring a human being. Some use workflows similar to those you encounter in a self-service phone system.

Chatbots are great project for students when restricted to a single subject unit. A student could program a chatbot with the terms from the current chapter in a science, history, or other textbook and then have the teacher "quiz" the chatbot on those concepts. Keeping in mind that programing a computer is like teaching an especially obtuse person, having a student "teach" a chatbot how to speak with a modicum of intelligence on a narrow subject requires a deeper exploration and internalization of that subject than memorizing facts for a test.

Board Game and RPG Game Frameworks

Gamification is a big deal in marketing and education right now. If you can turn a product into a game, customers will engage with it more fully. If you can teach children through games, they will immerse themselves more deeply in the subject. Every game your student coders build for your class and your school to educate their peers is a win, but building a game from scratch is hard. The programmer must solve physics, collisions, animations, distances, controls, and many more hard problems that a single researcher could pick just one of which and write a thesis on it.

Luckily we have Board Game and Role-Playing Game frameworks that comprise libraries of solutions to all these problems. They contain easy functions like onKeyDown() or onMouseClick() to handle keyboard and mouse inputs for controllng sprites on the game mat (craftyjs, 2016). They allow for setting up trigger() events that "listen" for events in the game and then execute an action in response to certain conditions being met. Most of these frameworks are for creating two-dimensional games, like board games that can run in a browser, but there are 3D engines as well for creating environments in which the user may have their hearing and vision completely augmented into a virtual world.

3D Computer Animation Frameworks

Javascript frameworks like three.js are making it easier than ever to produce mind-blowing interactive 3D computer animations in a web browser using the Web Graphics Library (WebGL) API. Computer animation frameworks allow for building complex 3D shapes, algorithms for generating geographies, a wide variety of animation types, allowing users to control the camera and explore the creations. There are thousands of open-source projects out there making use of this amazing invention righ this moment. The three.js website currently lists over 300 demos, including virtual earths, clouds of spinning cartoon monkey heads, virtual cloth blowing in a breeze, objects falling on rippling terrain, a torus made of flowing lava, morphing tea kettles, molecules for various compounds, and many other mind-blowing demos that are--quite honestly--horribly intimidating... that is... until you realize all the code is there for you to take an make your own.

One example of the power of three.js and WebGL that really struck me was a comparatively simple demo of the platform's "logarithmic depth buffer" (which I put in quotes because I'm not entirely sure what it is). By scrolling the mouse-wheel in and out, the app smoothly takes you from a dot labeled "microscopic 1um" out through "city-sized (10 km)," "planet-sized (12,000 km),"solar system-sized (50 Au)," "mind boggling (1,000 ligh years)," all the way to "... (100,000,000 light years)." It was like the iconic short film "Powers of Ten" by Charles and Ray Eames, which starts with a couple having a picnic in Chicago and then zooms out to view the cluster of galaxies before zooming back in all the way down to the interior of a single atom--except, this demo was interactive. The audience member, instead of passively observing the changing perspective, becomes the pilot--an explorer navigating fantastical worlds of scale. One inspiration for me in encouraging others to code is the possibility of seeing how others will expand upon and enhance this simple demonstration.

Virtual Reality Frameworks

For the past few years, gamers have been obsessed with news about virtual reality (VR) headsets that will put players into virtual worlds in which they can look around and interact with their surroundings. While these technologies are incredibly impressive, they also come with price tags in the hundreds of dollars. This presents a imposing financial barrier to most parents and students. I predict that, just as we saw information technology and the internet take years to trickle out to poor communities, it might be a long time before high-quality VR comes to low-income audiences.

But once again, it's open-source to the rescue! In 2014, two years after fans donated $2,437,429 to the development of the Occulus Rift VR headset (kickstarter, 2012), google researchers David Coz and Damien Henry realized they could make a pretty descent VR headset with a pizza box, two lenses, and a cellphone (Connexion, 2014). They dubbed the invention Google Cardboard, and made the plans available free online for anyone to build their own. Soon, numerous companies were selling pre-cut and assembled Cardboard kits at $10 to $15 each, meaning for less than the price of a single console video game students can be virtually exploring famous landmarks from all around the earth, watching movies on giant virtual movie screens, or viewing cultural artifacts from different angles (Lowensoh, 2014).

Just as three.js provides demos of what their software can do, Google provides a website of Chrome Experiments for Virtual Reality where developers can view amazing VR demonstrations of what's possible in a web browser. Students can ride a roller coaster, fly over a virtual city, or explore a geometric universe. Once again, the framework and demo code is there to take and make one's own.

Machine Learning Frameworks

While charting, gaming, and even 3D javascript frameworks are well-established and stable. I would describe Google's VR software as bleeding edge. It's buggy and advanced, but instructive to work in for all its challenges. Another software on the border between cutting and bleeding edge is Machine Learning, that is, giving computers the ability to learn without being explicitly programmed (Arthur Samuel, 1959). In other words, the program will learn how to identify patterns and solve problems by feeding it data, allowing it to formulate answers or find patterns, and then test those answers against a human teacher or through negative/positive reinforcement (Kohavi, 1998).

Machine learning is producing some incredible results. Computers are learning how to drive cars, play games, classify objects--without being told the categories before hand, and recognize patterns of meaning in speech (REFERENCE). At the same time, I must warn readers that I find machine learning HARD, and so do many other hackers. I have read many articles that are the equivalent to Machine Learning for Dummies, and I still only have the barest understanding of concepts like backpropagation, genetic algorithms, neural networks, and Bayesian networks. I have played with machine learning algorithms in JavaScript and C#, watching them produce outputs and perform math tricks that I am unable to truly appreciate.

This noted, I include Machine Learning frameworks here because I know many great minds who do understand and appreciate the technology. And I know that young brains will approach the subject open minds that are unencumbered with the preconceptions I suspect hinder my own understanding. If kids can understand machine learning, they might one day be able to explain it to adults like myself in terms we can understand.
